APPLICATION OVERVIEW

THE WEBSITE WITH THE ADMINISTRATION PANEL FOR THE MAINE COON CATTERY

My application is used in Maine Coon cattery in Silesia in Poland. It contains two parts - the website and the administration panel.
The website containing photos and information on running cattery and the animals helps future customers choose the right cattery and buy some animal. In addiction in this part you can find the location and the contact information and what's more you can use the contact form to contact with the cattery.
There is also administration panel for the breeder which helps them change some information on the website and check the messages sent by contact form. This panel has got also more tools. Firstly it contains the animal records, where the breeder can save a lot of important information about all animals and births. Secondly you can find there some customer and purchase records.
For more details, see the rest of the presentation.

THE WEBSITE PART


CHANGING NAVBAR

BIRTHS PAGE
User can decide how many births show up on the website. What's more main photo, amount and kind of details photos with all descriptions can be set and change in every moment by the breeder from the panel admin.
OUR CATS PAGE
As in the birth page user decides how many animals will appear on the website. Every photo and every description can be update in every moment by the panel. The animals are divided into two groups - men and women.
OPINION PAGE
The opinions are facebook plugins, it means that users are up to date with all likes and comments and they can be transferred to the Facebook with a single click.
CONTACT PAGE
This is the place with the contact form, which after completing the relevant data enables easy and quick contact with the breeder. All reports appear in the panel admin, which I'm going to show you in the next part. On this page are embed the cattery's fanpage and the Google map.

THE PANEL ADMIN PART


LOGIN
This is the place where the breeder can log in to the panel admin. The login data are inserted by the developer from the database. After successful login user can easily change from the website to the panel thanks to the customized navbar.
CONTACT FORM NOTIFICATIONS
All messages from the contact form the breeder can find in the list. The list can be filter by the name and the service status. The pagination and the choosing number of records per site are also available. In the details modal there is more information and there is the possibility to set the message as served if it's not.
CUSTOMERS
This is the place where the customers and the information about them are collected. In the filterable list there are all customers with the basic information. The available filters are first name, last name and city. Where there is more than 10 records the user can go to the next page or choose the bigger number of records per site. Adding new customer is done using the create modal, and after that you can edit it via the details modal. When the customer has got non-canceled transaction it isn't possible to delete that.
TRANSACTIONS
In this place there are all transactions, transaction means someone buys or reserves a cat. The table can be filter by customer first name, customer last name, animal name, animal lineage name, sale status and transaction status. Create modal allows adding new transaction and details modal allows updating them. The user can choose a customer and an non sold and unreserved animal from drop-down lists. While doing reservations, sales and cancellations the transaction status and animal sale status is changing.
ANIMALS
Firstly there is the list of animals. This can be filter by name, lineage name, gender, cattery status, sale status, website visibility status. Delete option doesn't work when the animal is connected with some transaction and it isn't cancelled.
Adding animals has got a few ways - for the cattery and for sale, so this has got many functions and possibilities. Some fields became disabled and set automatically depends on what the way the user choose and it can come back to the initial state when the breeder change the way. In this modal the user can choose the mother and the father from a drop-down list, and based on selections, the births results that also appear in a drop-down list are loaded. There is also the possibility to choose parents outside the cattery, then setting the birth date is required and setting the birth is disabled. What's more in both ways appropriate statuses are set. Setting the website visibility status has got another requirements depends on the website needing. There is also the possibility to add a photo with animal. The details modal works in a similar way but there existing animals are changed.
BIRTHS
As before, the list has a filtering option. The user can filter by birth name, mother and father. Pagination options are also similar. The amount of animals in a birth changes automatically when adding a cat to the birth. Adding birth is also via the modal when user has to set or choose from a drop-down list the appropriate data. Setting the website visibility status has got another requirements depends on the website needing. Uploading images is also available. Details modal is used for read more information, watch photos and update them. There is no way to delete the birth containing any animals.
